### P1 — 技术债 & 核心体验

- [x] **NumType 语义决策与落地** ✅ 2026-05-04
  - 路线 A（已存在）：`Mode.ParseToFraction()` 根据 `NumType` 选择解析策略 — Fraction→分数字符串解析，Decimal/Integer→double 解析
  - 路线 B（本次实现）：
    - `Fraction.ToString()` 智能格式化：分母质因数仅有 2 和 5 → 展示为小数；有其他质因数 → 展示为 `"num/den"` 分数
    - `Mode.FormatFractionResult(double)` 将 double 转回 Fraction 后智能格式化，无理数结果回退为普通小数
    - 8 个 double-based Mode 全部改用 `FormatFractionResult`
    - 示例：1/3 → `"1/3"`，3/10 → `"0.3"`，√2 → 保留小数
  - 涉及：`Fraction.cs`、`Mode.cs`、8 个 Mode 子类、`CalculateHelperBinaryTests.cs`（+2 新测试）

- [ ] **分数结果 RichText 上下标渲染**
  - 现状：计算结果中的分数以纯文本 `"num/den"` 展示（如 `"1/3"`）
  - 提案：用 Godot BBCode 将分子设为上标、分母设为下标，视觉上接近数学书写习惯（如 `[sup]1[/sup]/[sub]3[/sub]`）
  - 备选：Unicode 分数符号（仅覆盖少数种）、多 Label 分式布局（最重）
  - 涉及：`Fraction.ToString()` 或结果展示层的格式化逻辑

- [ ] **Poker 各状态视觉差异化**
  - 现状：Idle/UnSelect/OnSelect/Drag 四状态仅通过上下偏移区分（OnSelect 上移半高），其余状态无视觉差异
  - 提案：
    - OnSelect：发光边框 / 轻微放大（scale 1.05）/ 色调变化
    - Idle（禁用时）：灰度/半透明（当前完全看不出禁用）
    - UnSelect 悬停：色调微调（鼠标进入/离开已有事件，缺视觉处理）
    - Drag：原位保留半透明残影
  - 涉及：`Poker.cs`、各 State 类、着色器材质参数

- [x] **PokerFactory 参数化创建** ✅ 2026-05-05
  - `IPokerFactory` 新增 `Product(SuitType, string, NumType)` 重载
  - `PokerFactory` 实现：创建实例后直接设置花色、数值、数值类型
  - `CalculateMenu.CreateTest()` 简化为 `Deck.Add(PokerFactory.Product(suit, value))` 一行，消除 `AddCard` 辅助方法

- [x] **NumValue 输入验证** ✅ 2026-05-04
  - `Mode.IsValidNumValue(string, NumType)` 公共静态方法，包裹解析逻辑 + try-catch
  - `IPoker.IsValid { get; }` 属性，Poker 实现为 `Mode.IsValidNumValue(NumValue, NumType)`
  - Fraction 类型：校验 `"a/b"` 格式（两个有效 long，分母非零）
  - Decimal/Integer 类型：校验 `double.TryParse` 通过
  - PokerStub.IsValid 默认返回 true

- [ ] **PokerDragStartedEvent 监听者**
  - 现状：`DragState.Enter()` 发送此事件但无人监听
  - 可能用途：拖拽时高亮 Deck 可用卡槽、显示拖拽光标、临时禁用 Selector
  - 涉及：`DragState.cs`（已有发送）、`Deck.Events.cs` 或 `Selector`
